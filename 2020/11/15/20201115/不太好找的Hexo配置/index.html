不太好找的Hexo配置

发表于2020-11-15|更新于2020-11-15
|阅读量:

个人碰到的hexo上的配置问题

hexo 的配置文件有两个，文件名称都是 _config.yml

主目录里有一个，主题目录里有一个，第一次配置可以把两个配置文件过一遍。

以后为了方便可以，直接拷贝gists上面的配置文件备份。

根据文档中配置应该是最新的，但是总有一些问题找不到或者难以找到。

如何设置语言为中文?

语言可以在主题目录中看lang文件夹内有那些，然后进行设置

比如在主目录中配置：

lang:zh-CN

生效

keywords 设置是干什么的？

keyword早期是SEO重要的一个方面，如今淡化了，暂时不用设置。

标签页面报错 Cannot GET /tags/

需要执行命令生成

hexo new page tags

主题图片替换

web icon 是 32x32

用户图像是直接从github上同步的

配置各个页面的top-image

文章页面的top-image 用cover属性指定

评论配置

之前用过gitalk的评论，这次没有配好:pensive:

gitalk 的配置办法：在主题配置中，指明使用gitalk

在github中的develop设置里，配置一个oauth验证，并拿到ak、sk

最主要的是配置好 回调 callback_url，必须是博客地址，所以在测试的时候 localhost 是不对应的，总之搞复杂了，然后正好有其他的评论，还不如试一试

这次先用了 Valine 这个，文档 https://valine.js.org/quickstart.html

跟着教程，注册，设置，很容易就弄好，而且效果也比较好，嫌弃一下 gitalk :artificial_satellite:
