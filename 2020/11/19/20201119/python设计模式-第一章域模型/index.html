建模领域（Domain Model）

翻译不知道对不对。英文看得头疼，下有道云翻译了。

概念有：Entity（实体）, Value Object（值对象）, and Domain Service（域服务）

根据我浅薄的阅读能力， Domain Model 指的是一种能够从业务逻辑中，抽象出精髓的能力，另一种说法，就是建模。

当然，不可能建立出一个模型，能够覆盖所有的实际情况，但是，一个好的模型，应该能够解决当前的情况。

如何才能建立一个好的模型？

例子: 家具公司的出入库管理模型。

库存20个，卖了2个，还剩18个。

简单来说，能够实现这种逻辑的是模型，就行。

为了验证模型，文中讲了单元测试的方式，正好单元测试这部分比较菜，这里详细学习下。

Entity 和 Value Object

Entity 和 Value Object 是 DDD （demain driver design)的两个概念。

Entity表示的是，一个实体，即使改变了一些属性，仍然是之前的那个东西

Value Object 表示的是，值对象，通常改变后，就是另一个东西了。
